# 解决构建警告：统一采用命名导出 (Named Exports)

您终端中出现的警告（`using named and default exports together`）是因为 `Animation.ts`和 `random.ts` 同时使用了默认导出（`export default`）和命名导出（`export const/type`）。这在打包成 CommonJS/ESM 混合库时容易导致 `chunk.default` 访问问题。

为了消除警告并符合最佳实践（更有利于 Tree-shaking），我计划将它们**统一为命名导出**。

## 实施计划

### 1. 重构 `Animation.ts`

* **修改定义**：将 `export default class Animation` 改为 `export class Animation`。
* **更新引用**：
  * `packages/render/src/animation/Animator.ts`: 改为 `import { Animation } ...`
  * `packages/render/src/index.ts`: 改为 `export { Animation, Easing } ...`
  * `packages/render/src/animation/__tests__/Animation.test.ts`: 改为 `import { Animation } ...`

### 2. 重构 `random.ts`

* **修改定义**：删除 `export default uuidV4;`（保留现有的命名导出 `export const uuidV4`）。
* **更新引用**：
  * `packages/render/src/index.ts`: 改为 `export { uuidV4, getUnit32RandomValues } ...`

### 3. 验证

* 重新运行构建命令 `pnpm --filter hudx-render run build`，确认警告消失且构建成功。

此变更纯粹是代码风格和导出规范的调整，不会影响功能逻辑。
